# CY-Translator

一个本地运行的 PDF 划词翻译工具，调用你自己的 API，没有会员费。

## 功能

**通用**

- 隔离域：框选图表、公式区域后，划词自动跳过其中的噪声文字；也可以在隔离域内划词，不会选中域外的词；隔离域配置跟随 PDF 持久化，可以分享给他人
- 自动句补全：划取半句时自动扩展到完整句子边界
- 公式元素回捞：正文中嵌入的公式符号能被正确纳入选区
- 支持任意 OpenAI 兼容接口（OpenAI、DeepSeek、本地模型等）
- PDF 目录索引：自动提取书签，点击跳转，左侧可折叠面板
- PDF 笔记：在任意位置放置笔记图标，点击展开编辑，跟随 PDF 持久化，支持分享
- 大文件懒加载：按需渲染页面，打开大 PDF 不卡顿

**划词速翻**

- 可选松开即译，或右键手动触发
- 句级缓存：翻译过的内容自动缓存，再次划到直接调取，不重复消耗 token

**翻译持久化**

- 设计初衷是逐段选取、积累一份 Markdown 格式的翻译稿，方便日后直接查看
- 速翻更适合临时阅读，但同样会通过缓存将译文保留下来，避免重复翻译

## 使用前准备

需要自行编写一个 `.txt` 格式的系统提示词文件，否则模型不知道你要做什么。

项目附带了两个模板提示词，分别对应划词速翻和 Markdown 输出场景，可以直接使用，也建议根据自己的领域做调整，通常效果会好一些。

## 安装

```bash
pip install -r requirements.txt
python main.py
```

## 使用

### 打开 PDF

点击打开 PDF 按钮，每个 PDF 旁有一个 `...` 配置按钮，可以为该 PDF 单独指定翻译缓存、隔离域、笔记的配置文件路径，默认生成在软件所在目录的 `data` 文件夹下。

### 配置 API

在右侧填写 Base URL 和 API Key（部分接口无需 Key），点击"获取模型列表"可以自动拉取当前接口支持的模型；若获取失败，也可以手动填写模型名。注意：只有完成过成功的 API 请求或连接测试的模型名，才会被加入候选名单。

选择提示词文件（`.txt` 格式），没有提示词文件时模型不知道要做什么，项目附带了两个模板可以直接使用。

### 划词翻译

划词后自动翻译，或划词后右键手动点击翻译。

双栏 PDF 请在第一次使用时打开右上角的双栏模式开关，单栏和双栏的缓存相互独立，切换后不会共用。

### 隔离域

点击"框选隔离域"后，在 PDF 上框选一个矩形区域。从矩形外开始划词时，会自动跳过所有矩形内的内容；从矩形内开始划词时，只会选中矩形内的词。

点击"管理隔离域"可以拖动调整矩形大小和位置，或删除已有矩形。

### 笔记

点击笔记按钮或按 `Alt+N`，然后在 PDF 任意位置单击，放置一个笔记图标。点击图标展开文本框编辑内容，笔记跟随文件路径持久化保存。右键笔记图标可以删除（不可撤销）。

### 翻译持久化

- **New**：创建一个新的 Markdown 文件，需要指定保存路径和文件名（不含后缀）
- **Append**：在已有 Markdown 文件末尾追加，需要选择一个已有的 `.md` 文件

粘贴时去换行：开启后，粘贴内容时会自动去除 PDF 复制出来自带的换行符，但保留手动输入的换行；手动清洗按钮则会删掉所有换行符，包括手动输入的。

两种翻译模式下，翻译进行时，重复提交无效，需等待当前翻译完成。

## 关于缓存和配置文件

首次触发缓存、切换单双栏或使用隔离域时，软件会在自身所在目录生成对应的缓存文件和隔离域配置文件，以 PDF 文件名命名。

移动或重命名 PDF 后配置会失效，需要在选择 PDF 的界面手动重新绑定。

缓存文件和隔离域配置可以分享给他人，对方打开同一篇 PDF 并绑定配置后，图表区域的隔离域和已有的翻译缓存都可以直接复用。

如果用的人变得很多的话，或许可以有一个仓库或者网站用于共享配置文件，软件也可以相应的增加诸如“一键分享”之类的功能，以doi或者arXiv ID来唯一索引；
不过如何确保有质量的提交是一个问题，但单机使用体验，个人认为是比较良好了，至于配置共享生态，是以后的事了，说不定这个仓库只会默默无闻呢~

## 已知限制

- 扫描版（图片型）PDF 暂不支持，需要 PDF 内有可选中的文字层
- 公式密集区域的回捞效果取决于 PDF 本身的文字编码质量
- 最小单元是单词，是无法选中字母的，只测试过英文文献
